# 方剂谱系重建 (v0.2)

实现设计草案「创新点 2：从方剂组成检索升级为方剂谱系重建」。

## 数据信号

方書语料把谱系信息编码在两处，本模块都加以利用：

1. **标题层级嵌套**：`祖劑`、`湯頭歌訣`、`太平惠民和劑局方` 等以 `=` 数量表示
   层级（**越多 `=` 越外层**）：`======卷======`(6) ⊃ `=====門類=====`(5) ⊃
   `====方名====`(4) ⊃ `===子方===`(3)。在 `祖劑` 中，level-3 的子方嵌套于
   level-4 的祖方之下，直接给出「源方→衍生方」结构父子关系。

2. **行内加減线索**：`湯頭歌訣` 等以散文写明加减，如
   「益以半夏陳皮名**六君子湯**」「除卻半夏名**異功散**」「五苓散去桂為**四苓散**」。
   用正则 `(?:加|益以)…(?:名|為)(方名)` / `(?:去|除|除卻|減)…(?:名|為)(方名)`
   提取为 `Derivation(relation, herbs, target)`。

## 解析流程（`corpus/formula_loader.py`）

```text
split_sections_leveled  → (level, heading, body)
   ↓ 维护层级栈（弹出 level<=当前 的内层项，保留外层容器）
   category   = 最近的非方剂外层标题（門類/治法）
   parent_id  = 最近的方剂外层标题（祖方）
   ↓ 内容判定 is_formula：标题≤16字、非序跋、且 body 含≥2味已知药 或 含剂量模式
   组成 = 在「组成区」（去主治句、截至煎服法标记前）匹配的规范化药名
   主治 = 治/主治/療…句；煎服 = 上…/每服…/水煎… 段
```

药名匹配复用 v0.1 从本草语料构建的 **surface→canonical** 映射，故组成药味与
本草溯源、知识图谱、药对挖掘共享同一套规范名（黃耆→黃芪 等）。

## 谱系视图（`genealogy/FormulaGenealogy`）

给定方名，`genealogy(name)` 返回：

- **primary**：代表性版本——取各书复本中**众数组成**（对个别误抽稳健），故
  桂枝湯得到 桂枝/芍藥/生薑/大棗/甘草 而非某书残缺组成。
- **occurrences**：同名方跨书著录（历代演变时间线）。
- **ancestors / descendants**：沿结构父子链的源流与衍生方。
- **derivations**：行内加減记载。
- **similar（类方）**：组成 Jaccard 相似网络。用倒排索引 + 文档频率上限
  （跳过 `甘草` 等过common药做候选生成）控制复杂度，阈值 Jaccard≥0.34 且
  共享≥2 味，每方留 Top-8。

## 示例

```text
桂枝湯  组成 桂枝·芍藥·生薑·大棗·甘草
        衍生 桂枝加附子湯 / 桂枝加芍藥湯 / 桂枝加桂湯 / 桂枝新加湯 …
六味地黃丸  类方 都氣丸(0.875) / 八味丸(0.778) / 加減腎氣丸(0.75)
四君子湯  加減 加半夏陳皮 → 六君子湯
```

## 局限

- 个别书目（如 `千金`）组成排版差异大，单本组成可能不全；众数代表与跨书聚合
  缓解之。
- 结构父子忠于各书自身编排（如 `祖劑` 把一组水液方归于代表方之下），是「书内
  源流观」，非现代统一方剂学判定。
- 君臣佐使、剂量换算留待 v0.3。
